# W5500 이더넷 통신 안정성 및 자가 복구 (Network Reliability)

본 문서는 W5500(USR-ES1) 모듈의 SPI 인터페이스 설정과 네트워크 연결 유실 시의 자동 복구 메커니즘을 다룹니다.

---

## 1. 하드웨어 물리 계층 (PHY/Link)
- **SPI 인터페이스:** Pico의 한정된 핀 자원을 활용하기 위해 `SoftSPI` 모드를 사용하되, 통신 무결성을 위해 **100kHz 클럭 및 SPI Mode 3**로 최적화함.
- **전원 설계:** W5500의 높은 순간 전류(Peak Current) 대응을 위해 외부 3.3V 전용 레귤레이터를 배치하여 통신 중 보드 리셋 현상 원천 차단.

## 2. 자가 복구 메커니즘 (Self-Healing)
네트워크 장비나 게이트웨이(Pi 4)의 일시적인 장애 시 시스템이 수동 리셋 없이 자동 복구되도록 설계되었습니다.

- **연결 상태 모니터링:** 매 POST 요청 시 HTTP 상태 코드를 감시.
- **Fail Counter:** 3회 연속 전송 실패 시, 네트워크 스택이 꼬인 것으로 판단.
- **Hardware Reset:** W5500의 RST 핀을 트리거하여 칩을 초기화하고 DHCP부터 다시 수행하는 **전체 재초기화(Re-initialization)** 루틴 가동.

## 3. 데이터 전송 정책
- **게이트웨이:** 192.168.100.30 (Raspberry Pi 4)
- **프로토콜:** RESTful API (HTTP POST / JSON)
- **재시도 전략:** 일시적인 네트워크 노이즈에 의한 실패는 5초 주기의 다음 루프에서 자연스럽게 재시도됨.
