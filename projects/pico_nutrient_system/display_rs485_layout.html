📌 RS485 옵션 선택 시 진행
1. MAX485 모듈 #2 추가 구매 (양액 시스템 측에 설치)
2. Pico 빵판에 결선: 위 표대로 GP0/1/20을 MAX485 #2에 연결
3. RP2350 측: 내장 RS485 단자에 A/B 트위스트페어 연결, MAX485 추가 안 해도 됨
4. 종단저항 120Ω: 버스 양 끝 (Pico측 MAX485 + 디스플레이측)
5. Pico 코드: display_rs485.py 모듈 추가 (저분 fridge_controller 코드 기반)
6. 디스플레이 코드: 저분의 RP2350 통신부 그대로 가져다 씀 (포맷만 우리 시스템에 맞춰서)

🔁 두 옵션 동시 준비도 가능
• 이더넷 방식으로 가다가 문제 생기면 RS485로 폴백 가능 (펌웨어 스위치만으로 전환)
• 둘 다 빵판에 결선해두고 코드에서 선택 (다만 Pico 핀 다 점유됨)

⚠ 단점 (RS485 선택 시)
• 회사 DB 적재가 디스플레이에 의존하지 않음 → 큰 단점 아님 (Pico가 직접 DB POST)
• 디스플레이가 회사망 분리 → 원격에서 모니터링 못 함 (필요하면 Pico 거쳐서)
