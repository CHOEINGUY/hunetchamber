🌐 시스템 네트워크 토폴로지

W5500 이더넷 × 2개로 Pico W (양액 시스템) ↔ RP2350 (디스플레이) ↔ 회사 DB 연결

🌱 Pico W (양액 컨트롤러)

SPI0 → W5500
UART1 → RS485 센서
GPIO → 펌프/교반/LED/팬
역할: HTTP 서버 노출

🖥 RP2350 (터치 디스플레이)

SPI 외부 → W5500 (추가)
4" 480×480 IPS
역할: HTTP 클라이언트
1초 주기로 /api/status 폴링

🗄 회사 DB 서버

Pico → POST 적재
센서 시계열 + 이벤트
(디스플레이도 직접 조회 가능)

📌 통신 흐름
1. Pico → 회사 DB: 센서값/시퀀스 이벤트 주기적 POST (REST API).
2. 디스플레이 → Pico: GET /api/status 1초 폴링 → 화면 갱신.
3. 디스플레이 → Pico: 터치 시 GET /mix, GET /led?d=80 등 호출 → 동작 실행.
4. (옵션) 디스플레이가 회사 DB 직접 조회해서 그래프 등 표시 가능.

⚠ 미구현 / 확인 필요
• RP2350-Touch-LCD-4 보드의 외부 SPI 핀 확인 → W5500 결선 가능 여부 (LCD 내부 SPI와 별개).
• 디스플레이 측 코드: W5500 초기화 + HTTP 클라이언트 + UI 화면. 저분 RS485 통신 코드는 폐기.
• 회사 DB API 엔드포인트 (URL + 인증 키) 받기.

✅ 구현 완료
• Pico 측 main.py에 JSON API 추가: /api/status, /api/sensors, /api/recipe.
• RS485 센서 모듈 sensors.py (별 스레드 5초 폴링).
• W5500 SPI0 핀맵 + 회로도 (LAN 케이블만 꽂으면 됨).
