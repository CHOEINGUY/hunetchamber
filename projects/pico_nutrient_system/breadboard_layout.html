📌 배치 핵심
• Pico WH 방향: 보드가 길어서 USB를 좌측(또는 우측)으로 향하게 가로로 꽂음. USB를 위로 향하면 빵판 끝에서 튀어나옴.
• 핀 위치 (가로 꽂은 기준): 상단 = 핀 40~21 (USB 반대편), 하단 = 핀 1~20 (USB 쪽 행). 위 도면 상단 = VBUS/VSYS/3V3/...GP16, 하단 = GP0/GP1/.../GP15.
• 빵판 위: Pico WH (가로, 가운데) + W5500 (좌) + MAX485 (우). 신호선만, 12V 절대 X.
• 빵판 밖: 12V 어댑터, WAGO 3개 (3구×2 + 5구×1), LM2596, 4ch MOSFET 모듈 2개. 양면테이프로 평판에 고정.
• +12V 분배: 어댑터+ → WAGO+ #1 (3구) → 점퍼 → WAGO+ #2 (3구) → 1번/2번 MOSFET + LM2596 IN+.
• GND 분배: 어댑터− → WAGO− (5구 한 덩어리) → 1번/2번 MOSFET GND + LM2596 IN− (+ 1자리 예비).
• 빵판 GND 진입점: LM2596 OUT−에서 1가닥으로 → 빵판 GND 레일 (12V GND랑 자연 공통).
• 신호선: Pico GPIO → MOSFET S핀까지 점퍼와이어. 펌프(노랑) vs 교반/PWM(보라) 색 구분 추천.

⚡ PWM 운용 (SZH-AT021 포토커플러 한계: 1~2kHz)
• 양액 펌프 CH2/CH3: PWM 1kHz, 33% duty → 30 mL/min → 2mL 4초 (±2.5%)
• LED 스트립 (GP16): PWM 1kHz, 0~100% 디밍
• 팬 (GP18): PWM 1kHz, start kick 포함 (저듀티 stall 방지)
• 급수 CH1 / 출수 CH4 / 교반 GP11: ON/OFF (PWM 안 씀)

📡 RS485 센서 (Modbus RTU 9600 baud)
• 보유 센서 (저분과 동일): 온습도 addr 0x01 / 토양 5-in-1 addr 0x02 (수분/온도/EC/pH/N/P/K 한 센서) — 양액 EC/pH는 별도 센서.
• RS485 모듈은 자동방향: 4핀만 (VCC/GND/TX/RX). DI/RO/DE/RE 아님. Pico TX → 모듈 RX, Pico RX → 모듈 TX 교차.
• GP10은 이번 모듈에서 미사용 (자동방향이라 DE/RE 신호 안 들어감). 코드는 토글하지만 외부 영향 없음.
• 주의: 출고 기본 addr=1이라 멀티드롭 전 1개씩 addr 변경 필요 (tools_change_sensor_addr.py).

🔌 W5500 라벨 (절충형)
• MO = MOSI, MI = MISO, SCS = CS · SCK/RST/INT는 그대로
• 결선: VCC←3V3, GND, SCK←GP2, MO←GP3, MI→GP4, SCS←GP5, RST←GP6, INT→GP7

🔌 기타
• RS485 종단저항: 120Ω을 버스 양 끝(MAX485 측 + 마지막 센서 측)에 각각 1개.
• 팬 환류 다이오드: 1N5819 또는 1N4007 (캐소드=12V+, 애노드=팬 −).
• 전원 용량: 펌프+LED+팬 동시 피크 ~5A → 12V/5A 어댑터.
• 평판 사이즈: A4 (210×297mm) 정도면 충분. MDF나 아크릴판 추천.
