# 농장 센서 통합 및 RS485 통신 규격 (Sensors)

본 문서는 현장에 설치된 다중 센서의 RS485 Modbus RTU 통신 규격과 안정적인 버스 통합을 위한 물리 계층 설계를 다룹니다.

---

## 📡 1. RS485 버스 통합 설계 원칙
다양한 제조사의 센서를 하나의 버스에 통합하기 위해 아래 원칙을 엄격히 적용합니다.

- **통신 규격 통일:** 모든 센서는 **9600bps, 8-N-1**로 설정을 통일함. (기본값이 4800인 센서는 레지스터 수정을 통해 상향 조정됨)
- **Slave ID 관리:** 주소 충돌 방지를 위해 센서별 고유 ID(0x01~0x04)를 할당하고 소프트웨어 레벨에서 순차적 폴링(Sequential Polling) 수행.
- **전원 분리 및 접지:** 센서 구동 전원(12V SMPS)과 제어부(Pico) 전원을 물리적으로 분리하되, 신호 기준 전위 통일을 위해 **공통 GND(Common Ground)**를 형성함.

---

## 📂 2. 센서별 세부 사양

| 센서 항목 | Slave ID | 주요 레지스터 | 비고 |
| :--- | :--- | :--- | :--- |
| **온습도** | `0x01` | `0x0000`(습도), `0x0001`(온도) | 기본 기준 센서 |
| **토양 5-in-1** | `0x02` | `0x0000`~`0x0006` (수분, 온도, EC, pH, NPK) | 설정 레지스터 `0x07D0` 사용 |
| **조도(Solar)** | `0x03` | `0x0000` (일사량 W/m2) | 외부 12V 필수 |
| **CO2** | `0x04` | `0x0000` (CO2 ppm) | 예열 시간(약 2분) 고려 필요 |

---

## 🛡️ 3. 통신 신뢰성 확보 (Software Level)
- **UART Non-blocking Flush:** 요청 전 수신 버퍼의 잔여 데이터를 완전히 비워 데이터 오염 방지.
- **Adaptive Timeout:** 센서 응답 지연 시 전체 루프가 멈추지 않도록 300ms의 타임아웃 가드 적용.
- **CRC-16 Validation:** 수신된 모든 패킷에 대해 하드웨어 체크섬 검증을 수행하여 무효 데이터 제거.
