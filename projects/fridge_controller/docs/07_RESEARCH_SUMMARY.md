# 냉장고 정밀 온도 제어 시스템 연구 결과 요약 (Research Summary)

본 문서는 냉장고 제어 시스템 구축 과정에서 수행된 실험 데이터와 이를 바탕으로 수립된 제어 전략의 핵심 성과를 요약합니다.

---

## 1. 하드웨어 보호 설계 (Compressor Protection)
**"제어보다 보호가 우선"**이라는 원칙하에 컴프레셔의 물리적 소손을 방지하기 위한 알고리즘을 설계하고 검증했습니다.

- **연구 결과:** 컴프레셔 정지 직후 재가동 시 내부 압력 불균형으로 인해 기동 부하가 급증하는 현상 확인.
- **적용 로직:** 
  - **Min OFF Time (300s):** 냉매 압력 평형을 위한 최소 휴지 시간 확보.
  - **Min ON Time (300s):** 잦은 가동/정지(Chattering)로 인한 전력 낭비 및 모터 피로도 감소.
- **결과:** 실전 테스트 중 오작동이나 급격한 명령 변경에도 컴프레셔는 항상 안전 주기를 유지하며 구동됨을 확인.

## 2. 온도 하방 관성(Undershoot) 실측 분석
**"전원을 꺼도 온도는 계속 내려간다"**는 물리적 관성 현상을 실제 데이터를 통해 입증했습니다.

- **실험 조건:** 목표 온도 15.0°C / 히스테리시스 밴드 ±0.5°C (ON: 15.5°C, OFF: 14.5°C)
- **데이터 분석 결과:**
  1. **즉각적 냉각 반응:** 온도가 15.5°C에 도달하여 컴프레셔가 켜지는 즉시 온도가 하강하기 시작함 (냉각 응답성 양호).
  2. **오버런 현상(Undershoot):** 하한 임계값인 **14.5°C에서 전원을 차단했음에도 불구하고, 실제 내부 온도는 최대 13.7°C까지 추가 하강**하는 현상 확인.
  3. **운용 결과:** 결과적으로 시스템은 15.5°C ~ 13.7°C 사이에서 작동하며, **실제 평균 온도는 약 14.6°C(목표 대비 -0.4°C)**로 형성됨을 관찰함.

- **제어 최적화 및 최종 통찰 (Engineering Insight):**
평균 온도를 목표값(15.0°C)에 근접시키기 위해 두 차례의 시나리오 테스트를 거쳐 최적의 밴드 설정값을 도출했습니다.

| 시나리오 | 설정 (High / Low) | 실측 범위 (Max / Min) | 평균 온도 | 결과 분석 |
| :--- | :--- | :--- | :--- | :--- |
| **#1 대칭 제어** | +0.5°C / -0.5°C | 15.5°C / 13.7°C | **약 14.6°C** | 관성 하강으로 인해 목표보다 낮게 형성 |
| **#2 비대칭 제어** | **+1.1°C / -0.5°C** | 16.1°C / 13.8°C | **약 14.9°C** | **목표값(15.0°C)에 매우 근접 성공** |

- **최종 결론:** 
  1. 냉장고 컴프레셔 제어 시, 꺼진 후 발생하는 **하방 관성(Undershoot) 폭이 켜진 직후의 냉각 반응보다 훨씬 크고 길게 유지됨**을 확인.
  2. 따라서 원하는 평균 온도를 유지하기 위해서는 **상단 임계값은 대폭 높게(고온 방어), 하단 임계값은 상대적으로 좁게(과냉각 방어)** 설정하는 비대칭 전략이 필수적임.
  3. 이는 저장되는 내용물의 열용량(Heat Capacity)이나 보관 온도 가이드라인에 따라 유연하게 조정되어야 함.

## 3. 실시간 고해상도 모니터링 체계 (1Hz Logging)
산업용 수준의 정밀한 사후 분석을 위해 고해상도 데이터 파이프라인을 구축했습니다.

- **데이터 해상도:** 1초 주기의 HTTP POST 적재 (기존 농장 시스템 대비 5배 정밀도).
- **피드백 루프:** 게이트웨이 응답에 명령을 주입하는 'Response Injection' 방식을 통해 별도의 포트 개방 없이 양방향 제어 성공.
- **데이터 활용:** MariaDB에 적재된 시계열 데이터를 통해 컴프레셔 가동 시간 대비 냉각 효율(COP) 추정 가능.
