# 온도 제어 정책

## 안전 우선 순위

```text
1. forceoff (수동 강제 종료)
2. 컴프레셔 보호 (최소 ON/OFF 시간)
3. 자동 온도 제어 (히스테리시스)
4. 수동 명령
```

컴프레셔 보호는 항상 온도 조건보다 우선이다.  
온도 조건이 맞아도 보호 시간이 남아 있으면 대기한다.

## 컴프레셔 보호

| 규칙 | 설정값 |
|---|---|
| 최소 OFF 시간 | 300초 — OFF 이후 300초 안에는 켤 수 없음 |
| 최소 ON 시간 | 300초 — ON 이후 300초 안에는 끌 수 없음 |

이유: 컴프레셔를 빠르게 반복 켜고 끄면 기계적으로 손상될 수 있다.

## 온도 제어 (히스테리시스)

- 목표 온도 범위: 0.0 ~ 25.0 C
- 제어 폭: 목표 온도 ± 0.5 C

```text
upper = target + 0.5
lower = target - 0.5
```

냉장고가 꺼져 있을 때:

```text
sensor_temp >= upper  AND  last OFF 이후 300초 경과  →  켜기
```

냉장고가 켜져 있을 때:

```text
sensor_temp <= lower  AND  last ON 이후 300초 경과  →  끄기
```

그 외: 현재 상태 유지.

## Armed / Auto 상태

| 상태 | 의미 |
|---|---|
| `armed=0` | SSR 출력 차단 (안전 잠금) |
| `armed=1` | SSR 출력 허용 |
| `auto=0` | 온도 자동 제어 비활성 |
| `auto=1` | 온도 조건에 따라 켜고 끄기 시도 |

정상 자동 운전: `armed=1` + `auto=1`  
`auto 1` 명령을 보내면 `armed`도 자동으로 `True`가 된다.

## 기본값

| 항목 | 기본값 |
|---|---|
| 목표 온도 | 15.0 C |
| 제어 폭 | ± 0.5 C |
| 최소 OFF | 300초 |
| 최소 ON | 300초 |
| 팬 | 0% |
| LED | 0% |

## 명령 목록

```text
status              현재 상태 출력
auto 1 / auto 0     자동 제어 ON/OFF
arm / disarm        armed 상태 변경
on / off            수동 켜기/끄기
forceoff            즉시 OFF + disarm
target <0..25>      목표 온도 설정
fan <0|100>         팬 ON/OFF
led <0..100>        LED PWM (%)
minoff <초>         최소 OFF 시간 변경
minon <초>          최소 ON 시간 변경
```

## 센서 정책

현재 addr=1 단일 센서 사용. 두 번째 센서가 불안정하여 의도적으로 1개만 사용한다.  
센서 에러 발생 시 fail-safe 정책은 미구현 (다음 단계에서 결정 예정).
