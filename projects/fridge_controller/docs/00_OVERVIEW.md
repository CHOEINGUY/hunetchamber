# Fridge Controller — 프로젝트 개요

## 목적

냉장고 컴프레셔를 SSR로 제어하여 목표 온도를 유지한다. Pico WH가 온도를 읽고 제어 판단을 내린다.
상태는 DB에 기록되며, 웹 대시보드와 RP2350 터치 LCD로 확인·제어할 수 있다.

## 시스템 구성

| 구성요소 | 역할 |
|---|---|
| Pico WH (RP2040) | 센서 읽기, 온도 제어 판단, SSR 출력, Wi-Fi 전송 |
| RS485 온습도 센서 (addr=1) | 냉장고 내부 온도/습도 측정 |
| XSSR DA2410 | 220V 컴프레셔 라이브 라인 스위칭 |
| Raspberry Pi | HTTP 게이트웨이, MariaDB 저장, 웹 대시보드 |
| MariaDB | 운전 데이터 장기 저장 |
| RP2350 Touch LCD | 터치 리모컨 및 상태 디스플레이 |

## 제어 원칙

Pico가 최종 제어 판단을 내린다. RP2350과 웹 콘솔은 명령 전송 및 상태 표시만 한다.

Wi-Fi/DB 전송 실패가 냉장고 제어에 영향을 주지 않는다.  
이유: DB나 네트워크 장애가 냉장고를 멈추게 하면 안 되기 때문이다.

## 현재 상태 (2026-05-08)

- 전체 파이프라인 동작 확인 ✅
- Pico WH Wi-Fi → Raspberry Pi → MariaDB 전송 정상
- 220V 냉장고 전원 연결 후 실 운전 테스트 진행 중
- DB에 1초 주기로 데이터 적재 중

## 의도적으로 구현하지 않은 것

| 항목 | 이유 |
|---|---|
| 2센서 평균화 | 두 번째 센서가 불안정하여 현재 1개만 사용 |
| 센서 에러 fail-safe | 우선 동작 확인 후 정책 결정 예정 |
| PID 제어 | 히스테리시스로 충분한지 확인 후 검토 |
